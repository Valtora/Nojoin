# Browser Capture Guide

Nojoin records meetings directly from the browser.

## Supported browsers

- Chrome, Edge, Brave, and Arc on Windows for shared audio plus microphone
- Chrome, Edge, Brave, and Arc on Linux for shared audio plus microphone
- Chrome on Android and iOS for microphone-only recording

## Unsupported browsers and platforms

- Firefox
- Safari
- Other mobile browsers
- Chromium browsers on macOS

## How capture works

1. Start a meeting from Nojoin.
2. On desktop, pick the browser tab, window, or screen that contains the meeting.
3. On desktop, enable system or tab audio in the share picker when the browser offers it.
4. On mobile Chrome, grant microphone access and keep the phone near the meeting audio.
5. Nojoin uploads the recording in the background while the tab remains open.

## Setup tips

- Use the browser capture settings in Nojoin to pick the microphone and adjust gain.
- If microphone names are blank, allow microphone access once and refresh the device list.
- If a recording was paused, resume or discard it before starting a new meeting.
- On mobile Chrome, Nojoin records only the phone microphone; keep the Nojoin tab open and the phone awake.

## Troubleshooting

- If the browser says audio sharing is unavailable, pick a Chromium browser on Windows or Linux for shared audio, or use Chrome mobile for microphone-only recording.
- If levels look quiet, raise system or microphone gain in settings and retry with a short test meeting.
- If capture fails during sharing, start again and reselect the meeting source in the browser picker.
