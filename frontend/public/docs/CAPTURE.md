# Browser Capture Guide

Nojoin records meetings directly from the browser.

## Supported browsers

- Chrome, Edge, Brave, and Arc on Windows
- Chrome, Edge, Brave, and Arc on Linux

## Unsupported browsers and platforms

- Firefox
- Safari
- Mobile browsers
- Chromium browsers on macOS

## How capture works

1. Start a meeting from Nojoin.
2. Pick the browser tab, window, or screen that contains the meeting.
3. Enable system or tab audio in the share picker when the browser offers it.
4. Nojoin mixes shared audio with the selected microphone and uploads the recording in the background.

## Setup tips

- Use the browser capture settings in Nojoin to pick the microphone and adjust gain.
- If microphone names are blank, allow microphone access once and refresh the device list.
- If a recording was paused, resume or discard it before starting a new meeting.

## Troubleshooting

- If the browser says audio sharing is unavailable, pick a Chromium browser on Windows or Linux.
- If levels look quiet, raise system or microphone gain in settings and retry with a short test meeting.
- If capture fails during sharing, start again and reselect the meeting source in the browser picker.
